# CLAUDE.md тАФ starter-kit (meta)

**Important**: this file guides Claude when it **develops the starter-kit plugin**. It is **not copied** into user projects тАФ `skills/bootstrap/templates/CLAUDE.md.tpl` (and `CLAUDE.lean.md.tpl`) serve that purpose.

## Nature of the repo: two disjoint layers

This repo is both:

**Layer A тАФ Plugin sources** (what is published):
- `.claude-plugin/plugin.json` тАФ manifest
- `.claude-plugin/marketplace.json` тАФ mono-plugin marketplace for local install
- `skills/<name>/SKILL.md` тАФ logic of each `/starter-kit:<name>` slash command
- `skills/bootstrap/templates/*.tpl` тАФ Markdown templates used at generation time

**Layer B тАФ Project docs of the plugin itself** (self-application / dogfood):
- `README.md` тАФ public install and usage doc
- `CLAUDE.md` тАФ this file
- `docs/`, `brief/`, `docs/media/`, `PLAN.md` тАФ generated by the plugin applied to itself

The two layers **never overlap** at the path level. When you change something, ask yourself: "am I touching the plugin sources (A) or the project docs (B)?".

## Testing the plugin locally

```bash
# Install the local marketplace
claude
# then in Claude Code:
/plugin marketplace add /Users/guillaumeferrari/Projets/Starting-Claude
/plugin install starter-kit@starter-kit-local
```

Alternative for fast iteration:
```bash
claude --plugin-dir /Users/guillaumeferrari/Projets/Starting-Claude
```

Test from-scratch:
```bash
mkdir /tmp/test-bootstrap && cd /tmp/test-bootstrap
claude
# then: /starter-kit:bootstrap
```

Reloading after a change: a **new skill directory** requires a full restart of Claude Code; editing an existing `SKILL.md`/template is picked up by `/reload-plugins` (or `--plugin-dir` on the next launch).

## Template conventions

- Every template carries `<!-- generated-by: starter-kit v0.9.0 -->` at the top.
- Placeholders in `{{KEY}}` format (plain text substitution, no template engine).
- Available placeholders defined in `skills/bootstrap/SKILL.md` (Phase 5).
- **English-only**: one `.tpl` per file, no language suffix (cf. ADR 0012). `gitignore.minimal` and `adr-template.md` are plain files.

## Versioning

- Bump `version` in `.claude-plugin/plugin.json` AND `.claude-plugin/marketplace.json` at each release.
- Bump the `<!-- generated-by: starter-kit vX.Y.Z -->` signature in all templates consistently.
- Keep `CHANGELOG.md` up to date (Keep-a-Changelog).

## When to document

### ADR тАФ `docs/decisions/`
Any structural decision about the plugin (template format, new placeholder, new skill phase...) тЖТ an ADR.

### LEARNINGS тАФ `docs/LEARNINGS.md`
Claude Code plugin format gotchas, surprising `AskUserQuestion` behavior, `gh`/`glab` gotchas...

### PLAN.md
Ongoing tasks on the development of the plugin itself.

## Plugin workflow (mental model)

- **Template over code**: plain text `{{KEY}}` substitution in templates, never application logic nor a template engine (cf. ADR 0002).
- **Dogfooding**: the plugin applies to itself. `Starting-Claude/` contains its own layer B (docs/, brief/, PLAN.md...) generated by `/starter-kit:bootstrap`.
- **Offline-first**: only `/starter-kit:apply-best-practices` requires the internet (WebFetch). The rest works offline.
- **User handoff**: a generated `CLAUDE.md` is a **starting point**, not absolute truth. The user edits and enriches it per their project needs.

## Don't

- Don't put logic in `plugin.json` (just metadata)
- Don't make a single giant `AskUserQuestion` in a SKILL тЖТ group by theme (3-4 questions max per call)
- Don't generate an opinionated `.gitignore` from an unconfirmed stack тЖТ minimal by default
- Don't overwrite a user file without explicit confirmation (resume mode)
- Don't forget that `disable-model-invocation: true` hides the skill from auto-invocation: only the slash command works

## Tech stack

None. Pure Markdown + JSON. The "code" lives in the SKILL.md instructions that Claude executes on demand.
